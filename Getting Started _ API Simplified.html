Getting Started

Get started quickly by creating a payment in the sandbox, review in the test environment, then send to production.

This tutorial shows you how to use the Payment API to create a payment in the sandbox environment.

Complete these steps:

  1. Create a payment in the sandbox.
  2. Switch to the test environment.
  3. Switch to the production environment.

If you encounter any errors as you complete these steps, see our troubleshooting guide.

API Key and Authorization

Please note that the sandbox and production environments have different security requirements. While a key is adequate for accessing the sandbox, test and production environments require a bearer token or certificate.

Payment API tutorial

Step 1: Create a payment in the sandbox

All API calls must be authorized. In this step of the tutorial your requests will be authorized using an API key that grants you access to the sandbox APIs.

You may obtain your sandbox API key on your profile page.

Later you’ll learn how to authorize your API calls in the test and live environments using OAuth 2.0 credentials or client certificates. Only the sandbox environment uses API keys.
